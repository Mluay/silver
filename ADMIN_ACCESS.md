# روابط الموقع ولوحة التحكم

## روابط التطبيق (عند التشغيل المحلي)

| الصفحة | الرابط |
|--------|--------|
| **صفحة الزبون (المتجر)** | http://localhost:3000 |
| **لوحة التحكم (تسجيل الدخول)** | http://localhost:3000/admin/login |
| **لوحة التحكم (بعد الدخول)** | http://localhost:3000/admin |

---

## اسم المستخدم وكلمة المرور

**لوحة التحكم** تستخدم **Supabase Auth**. لا يوجد مستخدم افتراضي في المشروع.

### إنشاء مستخدم أدمن

1. افتح [Supabase Dashboard](https://supabase.com/dashboard) → اختر مشروعك.
2. من القائمة: **Authentication** → **Users**.
3. اضغط **Add user** → **Create new user**.
4. أدخل:
   - **Email**: البريد الذي تريد استخدامه للأدمن (مثال: `admin@silver-baghdad.com`)
   - **Password**: كلمة مرور قوية
5. احفظ البريد وكلمة المرور لديك.

### الدخول للوحة التحكم

- افتح: **http://localhost:3000/admin/login**
- أدخل **نفس البريد** و**كلمة المرور** اللذين أنشأتهما في Supabase.
- بعد تسجيل الدخول ستُحوَّل إلى لوحة التحكم.

---

## إذا ظهر «Invalid API key»

المصادقة تستخدم مفتاح **anon (public)** من Supabase وليس مفتاحاً آخر.

1. افتح [Supabase](https://supabase.com/dashboard) → مشروعك → **Settings** (أيقونة الترس) → **API**.
2. في **Project API keys** انسخ المفتاح المسمى **anon** أو **anon public** (يبدأ عادةً بـ `eyJ...`).
3. في المشروع ضعه في ملف `.env` أو `.env.local`:
   ```env
   NEXT_PUBLIC_SUPABASE_ANON_KEY="eyJ..."
   ```
4. أعد تشغيل السيرفر (`npm run dev`) ثم جرّب تسجيل الدخول مرة أخرى.

لا تستخدم مفتاح **publishable** أو **service_role** مكان مفتاح anon.

## ملاحظة

- **صفحة الزبون** لا تعرض أي زر لـ «لوحة التحكم»؛ الدخول لها يكون فقط عبر الرابط أعلاه.
- زر **الوضع الليلي/النهاري** في رأس صفحة الزبون يحفظ اختيارك تلقائياً.
